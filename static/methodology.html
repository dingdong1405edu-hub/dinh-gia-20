4.1 Thanh khoản

Tên tỷ sốCông thứcDiễn giải
Hệ số TT hiện hành (current_ratio) current_assets_total / current_liabilities_total Tài sản ngắn hạn ÷ Nợ ngắn hạn. >2 tốt, <1 báo động.

4.2 Đòn bẩy / Cơ cấu vốn

Tên tỷ sốCông thứcDiễn giải

Nợ / VCSH (debt_to_equity) total_liabilities / total_equity >2x là rủi ro với SME.
Hệ số nhân VCSH (equity_multiplier) total_assets / total_equity Đòn bẩy tổng tài sản trên VCSH.

Nợ / EBITDA (debt_to_ebitda) total_liabilities / operating_profit* * Khi BCTC không có khấu hao tách riêng, EBIT được dùng làm proxy cho EBITDA. Nếu BCTC có D&A, sửa ebitda_proxy trong file.

5. Agent 5 — Dự phóng 5 năm AI

Claude lập dự phóng KQKD + FCFF 5 năm. Là phán đoán nhưng bị ràng buộc bởi công thức ghi trong prompt. Kế toán có thể audit bằng cách kiểm tra từng năm có khớp công thức không:
FCFFyear = EBIT × (1 - tax_rate%) + D&A − CAPEX − ΔWC
EBITDAyear = EBIT + Depreciation
Gross Profityear = Revenue × Gross Margin
EBITyear = Gross Profit − OPEX (= Revenue × OPEX%)
Profit_Before_Tax = EBIT − Interest Expense
Net Income = Profit_Before_Tax × (1 - tax%)
Claude nhận BCTC + tỷ số + dữ liệu ngành rồi đề xuất 5 vector giả định: revenue_growth_pct[5], gross_margin_pct[5], operating_expense_pct_revenue[5], capex_pct_revenue[5], tax_rate_pct, depreciation_pct_revenue, working_capital_days. Mỗi giả định kèm rationale — đọc rationale trước khi tin con số.

6. Agent 6 — Định giá PYTHON + AI

Đây là phần quan trọng nhất để audit. Có 4 phương pháp + Sensitivity, được chạy bằng Python với giả định do Claude đưa.

6.1 DCF (Free Cash Flow to Firm) — PYTHON

Bước 1. Claude đề xuất WACC, terminal_growth (chiết khấu tốc độ tăng trưởng dài hạn).
Bước 2. Lấy 5 giá trị FCFF từ Agent 5.
Bước 3. Python chiết khấu về hiện tại theo công thức:
// Discount mỗi năm: PV(FCFF_t) = FCFF_t / (1 + WACC)^t // t = 1..5 // Tổng PV của 5 năm explicit: PV_explicit = Σ PV(FCFF_t) for t in 1..5 // Terminal value (Gordon Growth Model, áp dụng SAU năm thứ 5): Terminal_Value = FCFF_5 × (1 + g) / (WACC - g) // Yêu cầu WACC > g // PV của terminal value (chiết khấu về hiện tại): PV_Terminal = Terminal_Value / (1 + WACC)^5 // Enterprise Value: EV = PV_explicit + PV_Terminal // Equity Value: Equity_Value = EV - Total_Debt
Lưu ý quan trọng:
  • Total_Debt lấy từ balance_sheet.current.liabilities.total_liabilities (toàn bộ nợ phải trả, không chỉ nợ vay). Nếu kế toán muốn chỉ trừ nợ có lãi (interest-bearing), sửa tại file dưới.
  • Nếu có năm FCFF âm/null → Python trả về DCF rỗng kèm note.
  • Nếu WACC ≤ terminal_growth → terminal value không tồn tại (chia cho 0 hoặc âm) → Python trả null cho terminal.

📁 Sửa giá trị WACC / g do Claude đề xuất: agents/valuator.py · prompt trong _claude_assumptions (line 68)

📋 Mẹo audit nhanh cho kế toán

  1. Tải file Excel, vào sheet "Phương pháp tính" trước — nó có pointer đến file nguồn cho mọi phép tính.
  2. Sheet "Tỷ số": cột "Công thức" hiển thị trực tiếp công thức bên cạnh con số. So sánh với BCĐKT/KQKD ở 2 sheet đầu.
  3. Sheet "Định giá": block "DCF — Bước tính" cho thấy từng năm: FCFF → discount factor → PV. Cộng dồn ra EV.
  4. Sheet "Sensitivity": ô có viền/nền đậm hơn = base case. Đối chiếu với equity value ở sheet "Định giá".
  5. Số do AI đề xuất (badge tím) đều có rationale trong file trace JSON. Tải về và xem trường raw_response hoặc thinking của agent tương ứng.
  6. Muốn rerun với giả định khác (vd WACC 18% thay vì Claude đề xuất 14%): sửa hàm _claude_assumptions để hard-code, hoặc thêm endpoint nhận override.
Cấu trúc dữ liệu trong trace.json

File trace.json chứa input/output đầy đủ của mọi agent, bao gồm thinking và raw_response của LLM:

{
  "job_id": "...",
  "agent1_extract": { "financials": {...}, "thinking": "...", "raw_response": "..." },
  "agent2_industry": { "industry": {...}, "thinking": "...", ... },
  "agent3_business": { "business": {...}, ... },
  "agent4_ratios":   { "ratios": {...}, "growth": {...} },
  "agent5_projector":{ "projection": {...}, "thinking": "...", ... },
  "agent6_valuator": { "assumptions": {...}, "dcf": {...}, "multiples": {...},
                       "sensitivity": {...}, "summary": {...},
                       "thinking": "...", "raw_response": "..." },
  "agent7_thesis":   { "thesis": {...}, "thinking": "...", ... }
}

Trường thinking = chuỗi reasoning của Claude (extended thinking) trước khi đưa ra số liệu cuối. Đây là chỗ "vàng" để hiểu vì sao Claude chọn con số đó.
